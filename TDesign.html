Designing the Timeline interface:

After gaining insights on developing a tool to enhance the workflow of investigative journalists, we moved on to the design phase. Using Figma, we created the initial design with a focus on data analysis, specifically the "Data Item" page. This page showcases the results of data analysis, verifying and organizing data items chronologically. The goal is to utilize AI to analyze the dataset, extract key details like time and location, and present them in a user-friendly format.






Data item page/pop-up:

The image at the top is the design page for the Data-item page/pop-up, I will go through the six iteration that I went through. A brief explanation of the timeline: A program that allows investigative journalist to arrange a large data-set in a chronological order by using artificial intelligence to analyze the each data-item. This is done by multiple different AI-tools for example: object detection, scene detection, image-to-text and sentiment analysis from text. The AI tools can estimate the time, location and sentiment of each data-set this analysis will be shown through "estimation/verification" meter.

Input
The investigative journalist (IJ) uploads a dataset comprising images, videos, social media posts, and articles.
AI Analysis
The AI analyzes the dataset using multiple tools, including object detection, scene detection, image-to-text conversion, and sentiment analysis.
Chronological organisation
Once the AI analysis is complete, each data-item is arranged in a chronological order.
Verification/estimation meter
The timeline displays a green meter for each data-item, indicating the level of verification and estimation of the analyzed time, location, and sentiment.


Input and AI analysis

In the case of the corona curfew protest, there is a large data-set of links to social media posts. Finding out how these events unfolded and how the protest spread is crucial. As soon as the investigative journalist imported the acquired data-set, the AI instantly starts analyzing each and every item in the data-set. The AI uses object detection, scene detection and sentiment analysis from the data-set that consist out of the images, videos, articles and social media post. Together with the insights of the prelimenary research and the interview with Omroep Flevoland, I knew that it was possible to analyze the data-set with numerous AI tools and that it is important to show what the AI has analyzed in order to not have any biases. Allow the investigative journalist to draw the correct narrative for the investigation. Full transparency.






Chronological order

When the AI has analyzed the entire data-set, it is then arranged on a timeline in a chronological order. The purple bar at the bottom represents the timeline start to finish. Each "significant" data-item is shown on the timeline and each data-item also has a green bar. This green bar represents the prediction of the details of that data-item. This is merely an estimation/verification of the prediction of the AI and that fascilitates the need for full transparency.






Verification/estimation meter

As soon as the Investigative journalist wants to view a more details of a single data-item, they can click on the desired data-item to get a more in depth view of the data-set. This contains the verification/estimation meter as well. The meter can be adjusted by manually adjusting the data-item details. Next to that there is also a description of the data-item, which is in this case a social media post from Facebook. The AI also generates key takeaways from the article, this is done with the capabilities of a LLM(Large Language Model) that can analyze text and sentiment as well.
